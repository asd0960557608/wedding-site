## 1. 如何建立 Supabase

1. 前往 [Supabase](https://supabase.com/) 並登入。
2. 點選 `New project`。
3. 選擇 Organization，輸入專案名稱，例如 `wedding-site`。
4. 設定 Database Password。
5. Region 建議選擇離主要使用者較近的區域。
6. 建立完成後，進入專案 Dashboard。

## 2. 如何建立資料表

在 Supabase Dashboard 左側選擇 `SQL Editor`，建立新的 Query，貼上以下 SQL 並執行：

```sql
create table guests (
  id bigint generated by default as identity primary key,
  name text not null,
  phone text,
  attend_status text default 'pending',
  guest_count int default 1,
  meal_type text,
  note text,
  table_no text,
  created_at timestamp with time zone default now()
);
```

如果你啟用了 Row Level Security，請依照實際安全需求建立 Policy。測試階段可先建立允許前端新增、查詢與更新的 Policy：

```sql
alter table guests enable row level security;

create policy "Allow public insert"
on guests for insert
to anon
with check (true);

create policy "Allow public read"
on guests for select
to anon
using (true);

create policy "Allow public update"
on guests for update
to anon
using (true)
with check (true);
```

正式上線時，建議後台另外加上登入驗證，避免任何人都能修改桌號與出席狀態。

## 3. 如何設定 API Key

1. 進入 Supabase Dashboard。
2. 左側選擇 `Project Settings`。
3. 選擇 `API`。
4. 複製 `Project URL` 與 `anon public` key。
5. 打開 `js/supabase.js`，替換以下內容：

```js
const supabaseUrl = 'YOUR_SUPABASE_URL';
const supabaseKey = 'YOUR_SUPABASE_ANON_KEY';
```

替換成：

```js
const supabaseUrl = 'https://your-project-id.supabase.co';
const supabaseKey = 'your-anon-public-key';
```

## 4. 如何部署到 Vercel

### 方法一：使用 Vercel 網頁部署

1. 將 `wedding-site` 專案上傳到 GitHub。
2. 前往 [Vercel](https://vercel.com/) 並登入。
3. 點選 `Add New Project`。
4. 選擇你的 GitHub Repository。
5. Framework Preset 選擇 `Other`。
6. Build Command 留空。
7. Output Directory 留空或填入 `./`。
8. 點選 `Deploy`。

### 方法二：使用 Vercel CLI

先安裝 Vercel CLI：

```bash
npm i -g vercel
```

在 `wedding-site` 資料夾執行：

```bash
vercel
```

依照畫面提示完成部署。之後正式發布可執行：

```bash
vercel --prod
```

## 5. 如何綁定自己的網域

1. 進入 Vercel Dashboard。
2. 選擇你的婚禮網站 Project。
3. 點選 `Settings`。
4. 選擇 `Domains`。
5. 輸入你的網域，例如 `wedding.example.com`。
6. 按照 Vercel 顯示的 DNS 設定，到你的網域商後台新增紀錄。

常見設定：

```text
Type: CNAME
Name: wedding
Value: cname.vercel-dns.com
```

如果使用根網域，例如 `example.com`，Vercel 會提供 A record 或其他建議設定，請以 Vercel 畫面為準。

DNS 生效通常需要幾分鐘到 24 小時。生效後，Vercel 會自動配置 HTTPS 憑證。

## 客製化內容

你可以直接修改以下內容：

- 婚禮名稱：各 HTML 頁面的 `Yu & 134`
- 新人姓名：首頁的 `余為哲 & 林淯阡`
- 婚禮日期：`index.html` 與 `js/supabase.js` 的 `WEDDING_DATE`
- 婚禮地點：`index.html`、`guide.html` 的 `新竹・芙莉嘉`
- Google Map：`guide.html` 的 iframe `src`
- Banner 圖片：`css/style.css` 的 `.hero::before` 背景圖片網址
- 背景音樂：`index.html` 的 audio source
- 報名截止日：`js/supabase.js` 的 `REGISTRATION_DEADLINE`
- 後台密碼：`js/admin.js` 的 `ADMIN_PASSWORD`
- 相簿照片：`gallery.html` 的圖片網址

## 後台登入

後台頁面 `admin.html` 目前有前端密碼保護，預設密碼在 `js/admin.js`：

```js
const ADMIN_PASSWORD = 'yu1342027';
```

這是輕量保護，適合私人婚禮網站使用。若要更正式的安全性，建議改用 Supabase Auth 或 Vercel Serverless Function 保護後台 API。

## 功能整理

- 首頁：婚紗 Banner、新人姓名、倒數、婚禮資訊、地點、流程、導覽列、背景音樂、平滑動畫
- 加入行事曆：首頁與婚禮指引頁可下載 `.ics` 行事曆檔
- 報名登記：寫入 Supabase `guests` 資料表，並支援姓名 + 手機查詢後修改
- 報名截止日：2027 年 2 月 28 日 23:59 後自動停止送出與修改
- 桌次查詢：用姓名或手機查詢桌號與備註
- 婚禮指引：Google Map、停車、交通、飯店資訊
- 後台：登入保護、統計儀表板、搜尋、修改桌號、修改出席狀態
- 匯出資料：後台可匯出 CSV 與 Excel
- 桌次視覺化：後台用桌卡方式查看每桌賓客與人數
- 婚紗相簿：`gallery.html`
- 婚禮 Q&A：`qa.html`

## 注意事項

目前後台是前端靜態頁面，適合內部測試或低風險情境。正式使用時，建議增加管理員登入、限制 Supabase update policy，或改成 Serverless API 保護管理功能。
